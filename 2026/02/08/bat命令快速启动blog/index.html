在配置部署Hexo博客时，我发现配置Hexo时需要打开如下工具：

  • VS Code // Hexo Butterfly主题的自定义配置
  • Git Bash // 上传Hexo更改内容到GitHub page 进行托管和展示
  • Obsidian // 兼容markdown编辑器进行blog的撰写
    每次打开时，动作重复机械，因此，我采用bat命令进行自动化任务优化Hexo的启动。

  • 将txt后缀名修改为bat
  • 双击bat文件查看是否运行
